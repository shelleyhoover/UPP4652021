Queering the Map

GIS is a male-dominated industry. Open Street Maps found that events focused on engaging women in open street mapping:

  • Only an estimated 13% of OSM volunteers are women
  • Typical contributers add data about landmarks, roads, forests, railways, rivers and buildings. Women mappers are more likely to add services such as hospitals, childcare services, toilets, domestic violence shelters and women's health clinics.
  • Women mappers map issues that disproportionately affect women, such as sexual harassment, sexual violence in public spaces, or femicides
  • Regional differences in women mappers: for example, women mappers in Tanzania focused on adding water well paths.
    • So what is Open Street Maps?
  • OSM is a free, editable map created by users utilizing free data such as: manual surveying, local knowledge, GPS devices, and aerial photography.
  • Created as a reaction to the predominance of proprietary map data

    • It is estimated that currently 1 billion people live in unmapped areas

    Without maps, it is extremely difficult to dispatch humanitarian workers effectively, plan a vaccination campaign, or trace the spread of diseases, leaving vulnerable places without access to humanitarian aid and basic services.

    Humanitarian Open Street Maps
  • HOT engages volunteers to contribute to large mapping projects
  • So far, HOT OSM volunteers have collaboratively mapped an area home to more than 150 million people
  • World Bank, the United Nations, the American Red Cross, and over 250 organizations use HOT OSM data

    So, AGOL has a few options for this:

  • Citizen Problem Reporter
  • Crowdsource Manager
  • Survey 123

    • We will build a Survey 123 survey and embed it in a Dashboard

    Note, you are able to embed the survey into other applications as well

    This demo will be one option for your AGOL project

    AGOL Dashboards: designed to display multiple visualizations that work together on a single screens

    Demo, in three parts:

    1: Create Survey 123 Survey

    2: Create Webmap

    3: Create Dashboard & Add Survey 123 Survey to it

    Step-by-step, for reference:

    Step 1: Create Survey 123 Survey

    1. Go to Survey123 Site or navigate from AGOL home page (Apps drop down > Survey123)
    2. Choose survey type (I’d start with blank to get a feel, but go with your heart)
    3. Design your survey (Appearance, Content, Thank you screen)
    4. Save, Preview, & Publish!
    5. Click Collaborate Tab > Set “Share to Everyone” & copy link

    PSA: Make sure all layers are set to public.

    If you're still having issues after checking all layers, check that the form is set to to allow editing on public layers:

    Navigate to your content, find the Feature Layer (hosted) form and click the name.

    From there, you’ll be taken to the details page. Select settings

    Check the “Approve this layer to be shared with the Public”

    Navigate to your content, find the Feature Layer (hosted) form and click the name.

    Step 2: Create a web map

    1. Go to your content, 4 files will be created from the Survey123 form
    2. Identify survey feature layer, click ellipses for more options – “Add to New Map”
    3. Design + Save map + Set to Public

    Step 3: Create a dashboard

    1. Go to Apps dropdown > Dashboard (or Content > Create > Dashboard)
    2. Add Map (+). Select map you made from 123Form
    3. Add survey: (+) > Embedded Content
      1. Select Type: Feature. Pop-up will show, select web map layer>
      2. Add URL for form (Reminder to get form URL: Go to Survery123 > Click the form you created > URL will be available under the collaborate tab)
    4. Style your dashboard and add elements!
